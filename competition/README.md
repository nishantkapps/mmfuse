## Workflow

### Step 1: Prepare dataset splits
```bash
python competition/prepare_dataset.py --dataset dataset/sdata --output-dir competition/data
```

### Step 2: Extract trajectory ground truth
```bash
python competition/extract_trajectories.py \
    --dataset dataset/sdata \
    --sample-list competition/data/train_samples.csv \
    --output competition/data/train_ground_truth.csv

python competition/extract_trajectories.py \
    --dataset dataset/sdata \
    --sample-list competition/data/test_samples.csv \
    --output competition/data/test_ground_truth.csv
```

### Step 3: Generate sample submission
```bash
python competition/sample_submission.py \
    --test-samples competition/data/test_samples.csv \
    --output competition/data/sample_submission.csv
```

### Step 4: Evaluate locally
```bash
python competition/evaluate.py \
    --ground-truth competition/data/test_ground_truth.csv \
    --predictions competition/data/sample_submission.csv \
    --output results.json
```

## Hosting on EvalAI

### Step 5: Package the challenge zip
```bash
python competition/evalai/package_challenge.py \
    --data-dir competition/data \
    --output mira_challenge.zip
```

This reads your ground truth from `competition/data/`, generates dev/test annotation
splits, bundles everything with the config and templates, and produces a single zip.

### Step 6: Upload to EvalAI

1. Go to [https://eval.ai](https://eval.ai) and create a host account.
2. Navigate to **Host a Challenge** → **Create Challenge** → **Upload Zip**.
3. Upload `mira_challenge.zip`.
4. EvalAI will parse the YAML, create the phases/leaderboards, and deploy the
   evaluation script automatically.
5. After upload, review the challenge page in draft mode and toggle `published: True`
   when ready.

### Step 7: Share with participants

Once published, share the challenge URL. Participants will:
1. Register on EvalAI and join the challenge.
2. Download the training data and sample submission from the challenge page
   (upload these as challenge resources via the EvalAI host dashboard).
3. Submit CSV predictions through the EvalAI web interface or CLI:
   ```bash
   pip install evalai
   evalai set_token <your_token>
   evalai push <submission.csv> --phase <phase_id>
   ```

## Submission Format

Participants submit a CSV with columns: `sample`, `action`, `dx`, `dy`

```csv
sample,action,dx,dy
p045_m1_part1.wav,Start,0.50,-0.30
p045_m1_part2.wav,Go Here,2.10,0.80
...
```

## Metrics

| Track | Primary Metric | Description |
|-------|---------------|-------------|
| Track 1 | Macro F1 | Balanced F1 across 8 action classes |
| Track 2 | MAE | Mean absolute error of displacement (cm) |
| Combined | 0.60 × F1 + 0.40 × (1 - NormMAE) | Joint leaderboard score |
